双向串口通信：STM32(HAL)与OpenMV4

这是关于STM32（基于HAL库）与OpenMV4之间进行双向串口通信的记录，主要实现STM32向OpenMV发送控制指令，同时OpenMV能将处理后的结果返回给STM32，适用于两个MCU之间的简单交互

串口初始化设置

首先在STM32端配置串口，这里为了方便接收不定长的数据，使用了空闲中断配合接收回调函数的方式，当然也可以用普通的接收中断

接收数据解析

接收到数据后需要按照协议进行解析，这里在 openmv.c 中编写逻辑，定义了帧头 0x2C、0x12 和帧尾 0x5B，只有符合这个格式的数据包才会被处理

发送指令给OpenMV

STM32向OpenMV发送数据时，为了防止传输过程中丢包，这里采用了一个简单的循环多发几次的策略，格式是用 * 开头，& 结尾

初始化串口

OpenMV端使用Python开发，直接调用 pyb 库即可，注意波特率要和STM32对应

数据解析与回传

OpenMV这边主要负责两件事：一是把处理好的数据打包发给STM32，二是解析STM32发过来的指令

发送函数 Send_To_Mcu 将数据按照STM32那边的协议（0x2C, 0x12…）打包

接收函数 Receive_Prepare 则是一个状态机，用来提取 * 和 & 中间的数字，这里减去48是因为ASCII码的缘故

Uploading Your Project Files to GitHub: A Guide to Using Git

Hexo Tutorials

how to use Git?

Git基本操作介绍

  • 目录的切换
git bash输入 解释
pwd 显示当前终端会话的目录位置
ls 查看当前目录的所有文件
cd.. 上一级
cd 下一级的目录名 进入下一级
  • git
git 解释
git version 输出版本号
git config –global user.name 你的名字 配置全局用户名
git config –global user.email 你的邮箱 配置全局邮箱地址
git init 初始化
git add hello.txt 把文件加入git的版本控制系统中
git add . 或者 git add * .txt 把当前目录全部文件加入git的版本控制系统中,现在只是暂时保存
git commit 把上一步(add)的提交,此时进入了_vim终端编辑器_

​

​ vim编辑器使用

  1. 进入了vim ，默认是不能写东西的， 要按a或者i进入编辑模式
  2. 写本次提交的说明
  3. 退出 先按esc退出编辑模式 再输入英文冒号: w q 保存 退出
git 解释
git log 查看**日志 ** git log –oneline 打印每行描述
git commit -m 第二次提交 写说明的简单操作 跳过vim ，提交后系统打印一串commit id
git reset –hard 粘贴commit id 回退版本 hard硬重置（覆盖所有变更、soft 、mixed
git branch 分支名 git commit的时候创建一个分支(git init之后不提交是分支不了的)
git branch -a 查看所有分支
git checkout 分支名 切换分支
git checkout -b user分支名 创建并切换到此user分支
git branch -d user分支名 删除分支
git merge user 合并分支: 切换到master 写git merge order ，有冲突会报 ，去对应文件修改，保存后add 、commit 尚硅谷新版Git快速入门(3h迅速掌握git)_哔哩哔哩_bilibili
  • 更多命令行介绍
git 解释
git clone 需要下载下来的仓库链接 remote-git-resp 将远程仓库的内容完全下载到本地的remote-git-resp文件夹
git config user.name 你的名字英文 配置当前仓库的用户名
git config user.email 你的邮箱 配置当前仓库的邮箱
git status 查看暂存区状态
git rm –cached a.txt 删除暂存区的文件
git restore a.txt 误删除后未提交,恢复误删除文件(之前commit过的)
git revert 误删文件后上一个版本号 还原：硬重置后,会回到最开始的提交,这样提交过程就消失了;所以此方法恢复当前版本，就要粘贴下一版本号内容,不丢失之前提交信息
git tag 标签名称 版本号 给当前提交版本加上标记==起别名：git tag uptfile 版本号 之后 再写git tag，之后git log uptfile标签名称 就可以查看历史信息(取代git log _版本号_的不方便查找) 可以给每个版本都添加标签但是标签不能重复
git tag -d 标签名称 删除标签
git checkout -b 标签名称 对当前标签所在的提交进行分支创建

从远程仓库克隆下来的仓库，”origin”来代替远程仓库-地址

git remote add origin github仓库地址 增加远程仓库地址，别名为origin
git remote remove origin 删除远程仓库地址
git remote rename origin 改名
