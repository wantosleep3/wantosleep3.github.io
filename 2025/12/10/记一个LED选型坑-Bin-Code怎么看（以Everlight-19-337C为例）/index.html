记一个LED选型坑 Bin Code怎么看（以Everlight 19-337C为例）

最近在做RGB灯效，选了亿光的19-337C/RSBHGHC-A01/2T。画板子前得看清Bin Code，不然容易踩坑。这是一颗常见的SMD RGB LED，我也借此记录下怎么看参数。

什么是Bin Code

灯珠生产没法做到完全一样。同一批出来的货，亮度、颜色、电压都有偏差。厂家把参数相近的挑出来放一起，这个动作叫Binning。那个分类代码就是Bin Code

不注意这个，买两卷灯带可能一黄一绿，或者亮度看起来一深一浅

翻看规格书

规格书地址 https://www.everlighteurope.com/custom/files/datasheets/DSE-0009219.pdf

这颗料是RGB三合一，打开规格书第4页。重点看红(RS)、蓝(BH)、绿(GH)这三个部分

亮度分档

肉眼看亮度最明显。规格书把亮度分了几档。

红灯(RS)
S1档 180到225 mcd
S2档 225到285 mcd
T1档 285到360 mcd

绿灯(GH)
U1档 450到570 mcd
U2档 570到720 mcd
V1档 720到900 mcd

蓝灯(BH)
Q1档 72到90 mcd
Q2档 90到112 mcd
R1档 112到140 mcd

你看绿色的亮度远高于蓝色，这是材料决定的。写代码调白光的时候别给一样的PWM。全给100%占空比，出来的光肯定发绿。

颜色波长

人眼对绿色很敏感。波长稍微差一点，就能看出翠绿和黄绿的区别。

这颗料的红光和蓝光没细分。绿光(GH)专门分了两档。
Y档 525到530 nm
Z档 530到535 nm

做屏幕或者背光一定要买同一个Bin，混着用会有色差。

货到了怎么认

看卷盘上的标签，参照规格书第9页的说明。

CAT 代表亮度
HUE 代表颜色波长
REF 代表电压

如果标签写着 CAT S2，HUE -，REF 2.0。意思就是这盘红灯亮度在225到285 mcd之间

软件怎么补

买不到同一个Bin怎么办？只能靠软件补。

比如一批S1较暗，一批T1较亮。硬件贴完了改不了，代码里就留个系数。针对不同批次调整PWM最大值。

几点经验

下单尽量只买同一个Bin。
收货核对标签，别混料。
写驱动留点余量，别把亮度写死
